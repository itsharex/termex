# V0.15.0 — Terminal Search System

## Context

终端文本搜索是所有主流 SSH 客户端的标配功能。基于对 iTerm2、Tabby、Warp、Windows Terminal、Termius、MobaXterm、Royal TSX、SecureCRT 等 8 款客户端的调研，本迭代实现完整的搜索系统，分三个阶段：

1. **终端内文本搜索** — 基线功能，8/8 客户端均支持
2. **持久化关键词高亮** — 高端差异化功能（iTerm2 Triggers / SecureCRT Keyword Highlighting）
3. **跨 Tab 搜索** — 效率提升功能，面向多会话运维场景

### 竞品调研摘要

| 特性 | iTerm2 | Tabby | Warp | Win Terminal | MobaXterm | SecureCRT |
|------|--------|-------|------|-------------|-----------|-----------|
| 终端内搜索 | Yes | Yes | Yes | Yes | Yes | Yes |
| 快捷键 | Cmd+F | Cmd/Ctrl+F | Cmd+F | Ctrl+Shift+F | Ctrl+F | Ctrl+F |
| UI 形式 | 浮动栏(右上) | 浮动栏(顶部) | 浮动面板 | 下拉栏(顶部) | 工具栏 | 模态对话框 |
| 正则表达式 | Yes | Yes | Yes | No | Yes | Yes |
| 大小写切换 | Yes | Yes | Yes | Yes | Yes | Yes |
| 全词匹配 | No | Yes | Yes | No | Yes | Yes |
| 高亮所有匹配 | Yes | Yes | Yes | Yes | Yes | 可选 |
| 匹配计数 | No | Yes | Yes | No | No | No |
| 关键词持久高亮 | Yes(Triggers) | No | No | No | No | Yes |
| 跨 Tab 搜索 | No | No | No | No | No | No |

### 技术优势

Termex 使用 xterm.js，`@xterm/addon-search` 开箱提供正则、大小写、全词匹配及装饰器高亮 API，核心搜索引擎零成本接入。

## Phase 1: 终端内文本搜索

### 1.1 搜索栏 UI 设计

浮动搜索栏，定位于终端右上角（与 iTerm2 / VS Code 一致），半透明背景 + 毛玻璃效果：

```
┌─────────────────────────────────────────────────────────────────────┐
│ Terminal content...                                                 │
│                          ┌──────────────────────────────────────┐   │
│                          │ 🔍 [Search...      ] [Aa][.*][W]    │   │
│                          │              3 of 42   [↑] [↓] [✕]  │   │
│                          └──────────────────────────────────────┘   │
│                                                                     │
│ $ ls -la                                                            │
│ drwxr-xr-x  5 user group 160 Mar 31 10:00 .                       │
│ -rw-r--r--  1 user group 256 Mar 31 10:00 file.txt                 │
└─────────────────────────────────────────────────────────────────────┘
```

**搜索栏元素**：
- 搜索输入框（自动聚焦，占据主要宽度）
- `[Aa]` — 大小写敏感切换按钮（toggle）
- `[.*]` — 正则表达式切换按钮（toggle）
- `[W]` — 全词匹配切换按钮（toggle）
- 匹配计数显示：`"3 of 42"` 或 `"No results"`
- `[↑]` — 上一个匹配（也可 Shift+Enter）
- `[↓]` — 下一个匹配（也可 Enter）
- `[✕]` — 关闭搜索栏（也可 Escape）

**交互规则**：
- `Cmd+F` / `Ctrl+F` 打开搜索栏，焦点自动进入输入框
- 若搜索栏已打开，`Cmd+F` 全选输入框内容以便替换搜索词
- 输入时实时搜索（incremental search），每次按键自动 `findNext`
- `Enter` → findNext，`Shift+Enter` → findPrevious
- `Escape` → 关闭搜索栏，焦点回到终端
- 切换搜索选项（Aa/.*/ W）时自动重新搜索
- 关闭搜索栏时清除所有高亮装饰

### 1.2 匹配高亮样式

使用 `@xterm/addon-search` 的装饰器 API：

```typescript

  matchBorder: "#FFD70060",

  activeMatchBackground: "#FF8C00A0",   // 当前匹配 — 橙色高亮

  activeMatchColorOverviewRuler: "#FF8C00",

```

### 1.3 快捷键绑定

| 快捷键 | 动作 | 范围 |
|--------|------|------|
| `Cmd+F` / `Ctrl+F` | 打开/聚焦终端搜索栏 | 全局 |
| `Enter` | 下一个匹配 | 搜索栏聚焦时 |
| `Shift+Enter` | 上一个匹配 | 搜索栏聚焦时 |
| `Escape` | 关闭搜索栏 | 搜索栏聚焦时 |

## Phase 2: 持久化关键词高亮

### 2.1 功能描述

用户可定义关键词规则列表，终端输出中匹配的文本会**自动持久高亮**，无需手动搜索。典型用途：

- `ERROR` → 红色背景高亮
- `WARNING` → 黄色背景高亮
- `SUCCESS` / `OK` → 绿色背景高亮
- `FAIL` / `CRITICAL` → 红色加粗
- 自定义 IP 正则 → 青色高亮

### 2.2 关键词规则模型

```typescript
interface KeywordRule {
  id: string;                // uuid
  pattern: string;           // 匹配模式（文本或正则表达式）
  isRegex: boolean;          // 是否为正则
  caseSensitive: boolean;    // 大小写敏感
  foregroundColor: string;   // 前景色（空则不改变）
  backgroundColor: string;   // 背景色
  enabled: boolean;          // 启用/禁用
}
```

**预设规则建议**（用户首次使用时可一键添加）：

| 模式 | 正则 | 背景色 | 前景色 |
|------|------|--------|--------|
| `ERROR` | No | `#EF444440` (红) | — |
| `WARNING\|WARN` | Yes | `#EAB30840` (黄) | — |
| `SUCCESS\|PASSED` | Yes | `#22C55E40` (绿) | — |
| `FAIL\|FATAL\|CRITICAL` | Yes | `#EF444460` (红) | `#FFFFFF` |

### 2.3 高亮实现方案

使用 xterm.js **缓冲区扫描 + 装饰器 API** 方案：

1. **初始挂载**：终端挂载时，扫描已有缓冲区内容，为匹配文本注册装饰器
2. **实时高亮**：监听 `terminal.onWriteParsed` 事件，每次新数据写入后扫描新增行
3. **增量扫描**：维护 `lastScannedLine` 指针，仅扫描新增内容，避免全量重扫
4. **规则变更**：规则增删改时，清除所有装饰器并全量重新扫描
5. **性能保护**：单个规则最大装饰器数量限制（默认 2000），超出后停止该规则的新增匹配

```
数据流：
SSH Data → terminal.write() → onWriteParsed → scanNewLines() → registerDecoration()
                                                    ↑
                                            keywordRules[]
```

### 2.5 数据持久化

关键词规则存储在 settings 数据库中，key 为 `keyword_highlight_rules`，value 为 JSON 字符串：

```json
[
  {
    "id": "uuid-1",
    "pattern": "ERROR",
    "isRegex": false,
    "caseSensitive": true,
    "foregroundColor": "",
    "backgroundColor": "#EF444440",
    "enabled": true
  }
]
```

**加载时机**：`settingsStore.loadAll()` 时解析，存入 `keywordRules: ref<KeywordRule[]>([])`。
**自动持久化**：`watch(keywordRules, ...)` 深度监听，变更时序列化回数据库。

## Phase 3: 跨 Tab 搜索

### 3.1 功能描述

在所有已打开的终端 Tab 中搜索文本，结果按 Tab 分组展示，点击结果跳转到对应 Tab 并定位到匹配行。面向多会话运维场景（同时开 5-10 台服务器排查问题）。

### 3.2 跨 Tab 搜索对话框 UI

通过 `Cmd+Shift+F` / `Ctrl+Shift+F` 打开模态对话框：

```
┌──────────────────────────────────────────────────────────────┐
│  Search All Tabs                                        [✕]  │
│                                                              │
│  [🔍 Search...                   ] [Aa] [.*] [W] [Search]   │
│                                                              │
│  ── web-server-01 (3 matches) ─────────────────────────────  │
│  │ L  42 │ 2024-03-31 ERROR: Connection timeout to db01     │
│  │ L 108 │ ERROR: Failed to read config file                │
│  │ L 256 │ WARNING: Retry limit reached, ERROR count: 5     │
│                                                              │
│  ── db-server-02 (1 match) ────────────────────────────────  │
│  │ L  15 │ ERROR: Replication lag exceeded threshold         │
│                                                              │
│  ── monitoring (0 matches) ────────────────────────────────  │
│  │ (no matches)                                              │
│                                                              │
│  Total: 4 matches in 2 tabs                                  │
└──────────────────────────────────────────────────────────────┘
```

**UI 元素**：
- 搜索输入框 + 搜索选项（Aa / .* / W），与 Phase 1 保持一致
- `[Search]` 按钮触发搜索（或 Enter）
- 结果按 Tab 分组，每组显示 Tab 名称和匹配数
- 每条结果显示行号 + 行内容（匹配文本高亮，上下文截断到 ~100 字符）
- 点击结果行 → 切换到对应 Tab + 打开该 Tab 的搜索栏 + 滚动到匹配位置
- 底部显示总匹配数和匹配 Tab 数
- 无匹配 Tab 默认折叠，可展开

**交互规则**：
- 搜索为手动触发（点击 Search 或 Enter），非实时搜索（避免大缓冲区卡顿）
- 搜索过程中显示 "Searching..." 进度指示
- 每个 Tab 结果上限 100 条，超出显示 "... and N more matches"
- 对话框关闭不影响各 Tab 的搜索状态

### 3.3 终端实例注册表

为实现跨 Tab 搜索，需要从外部访问各终端实例的缓冲区。创建全局终端注册表：

```typescript
// src/utils/terminalRegistry.ts
const registry = new Map<string, Terminal>();

export function registerTerminal(sessionId: string, terminal: Terminal): void;
export function unregisterTerminal(sessionId: string): void;
export function getTerminal(sessionId: string): Terminal | undefined;
export function getAllTerminals(): Map<string, Terminal>;
```

**注册时机**：`useTerminal.mount()` 中创建 Terminal 后注册，`dispose()` 中注销。

### 3.4 缓冲区扫描算法

```typescript
interface CrossTabMatch {
  sessionId: string;
  tabTitle: string;
  lineNumber: number;       // 缓冲区中的绝对行号
  lineContent: string;      // 行完整文本
  matchStart: number;       // 匹配起始列
  matchLength: number;      // 匹配长度
}

function searchAllTabs(
  term: string,
  options: { regex: boolean; caseSensitive: boolean; wholeWord: boolean },
  maxPerTab: number = 100,
): CrossTabMatch[];
```

**扫描逻辑**：
1. 遍历 `getAllTerminals()`
2. 对每个终端，遍历 `buffer.active.getLine(y).translateToString()`
3. 对每行执行匹配（支持正则/文本/全词/大小写选项）
4. 收集匹配结果，每 Tab 限制 `maxPerTab` 条
5. 使用 `setTimeout` 分批处理避免阻塞 UI

### 3.5 跳转到匹配位置

点击搜索结果后：
1. `sessionStore.setActive(sessionId)` — 切换到目标 Tab
2. 触发目标 Tab 的 TerminalSearchBar 打开，填入搜索词
3. 调用 `terminal.scrollToLine(lineNumber)` 滚动到匹配行
4. 使用 SearchAddon 的 `findNext()` 从当前位置开始定位到精确匹配

## 七、实施步骤

### Step 1: 基础设施（Foundation）

**目标**：安装依赖、创建类型定义、终端注册表

1.1 `package.json` — 添加 `@xterm/addon-search` 依赖
1.2 `src/types/search.ts` — 创建所有类型定义（SearchOptions、KeywordRule、CrossTabMatch 等）
1.3 `src/utils/terminalRegistry.ts` — 创建全局终端注册表（register/unregister/get/getAll）
1.4 `src/composables/useTerminal.ts` — 加载 SearchAddon、注册终端到 Registry、暴露 getSearchAddon() 和 getTerminal()

### Step 2: 单 Tab 搜索逻辑

**目标**：封装搜索状态管理
